The canonical external base URL of this server, e.g. https://jellyfin.example.com. When set, the redirect URI and SAML base URL are built from it instead of the request host, so a spoofed or proxy-forwarded host cannot redirect the login elsewhere. It overrides the scheme and port overrides above.
⚠ Recommended. When left blank, the redirect URI derives from the request Host header; behind a reverse proxy that is only safe if Jellyfin's Known Proxies is set and the proxy sends a trusted X-Forwarded-Host (not a client-supplied one) — setting this override is the robust fix. This is the base URL, not the redirect URI — register <this URL>/sso/OID/redirect/<ProviderName> (or the legacy /sso/OID/r/<ProviderName>) as the exact redirect URI at your provider.
